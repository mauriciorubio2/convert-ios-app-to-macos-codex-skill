# Contributing

Thanks for improving this skill.

## Guidelines

- Keep changes focused on iOS-to-macOS conversion workflows.
- Prefer concise skill instructions and move longer examples into `references/`.
- Do not add app-specific secrets, credentials, API keys, customer data, or private release identifiers.
- Update tests or docs when changing behavior.
- Run validation before opening a pull request.

## Pull Requests

Include:

- Elevator pitch / summary of the change.
- Why it helps future conversions.
- Validation commands and results.
- Any remaining risks or known gaps.
